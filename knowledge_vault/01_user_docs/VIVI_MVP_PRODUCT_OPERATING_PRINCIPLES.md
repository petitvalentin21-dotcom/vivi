# VIVI MVP — Principes produit opératoires

## Vision active

VIVI est un assistant local-first: "je parle, VIVI répond".

## Choix de simplicité

- interface web unique ;
- backend API local ;
- LM Studio unique provider MVP ;
- RAG lexical explicable ;
- sources visibles.

## Non-objectifs volontaires

- plateforme d'agents ;
- routage multi-provider ;
- autonomie documentaire non supervisée ;
- complexité cloud.

## Différenciation vs legacy VIVI_IA

- VIVI est un produit clean centré MVP ;
- VIVI_IA sert d'archive d'inspiration ;
- pas d'import massif legacy ;
- décisions orientées exploitabilité réelle.

## Roadmap implicite actuelle

1. conserver la stabilité MVP validée ;
2. améliorer qualité documentaire et retrieval ;
3. renforcer runbooks/tests/sourçage ;
4. préparer post-MVP sans implémentation prématurée.
